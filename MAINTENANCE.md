# RCV Lab Website Maintenance Guide

This site is a React + Vite single-page app. Content is statically served from `public/` and loaded at runtime via `fetch`. No server or database is required.

## Running the project
- Install deps: `npm install`
- Dev server: `npm run dev` (open http://localhost:5173/rcv_website/ because `base` is `/rcv_website/`)
- Build: `npm run build`

## Where member data lives
- Profiles: `public/content/authors/<Name_Id>/_index.md`
- Avatars: `public/content/authors/<Name_Id>/avatar.png` (default image: `public/assets/media/authors_research/default_avatar.png`)
- People page config (groups/order): `public/content/people_page/people.md`
- Author index consumed by the app: `public/assets/data/authors.json` (generated by scripts)

Each `_index.md` frontmatter should include `title`, `role`, and `user_groups` (e.g., `Ph.D. Students`, `M.Sc. Students`, `Undergraduate Students`). `weight` controls ordering (lower appears first). Body sections use `### About Me`, `### Research`, and `### Publications`.

## Adding members (recommended workflow)
1. Update `scripts/test-members-data.json` with the new member entry or create a similar dataset.
2. Run `npm run members:add-test` to generate the profile folder, copy the avatar placeholder, and refresh `authors.json`.
3. Replace `avatar.png` with the actual photo if available.
4. If you edit profiles manually, run `npm run members:sync-index` afterward to refresh `public/assets/data/authors.json`.

## Removing the temporary test members
- `npm run members:remove-test` deletes the generated test profiles and regenerates `authors.json`.

## Verification checklist
- `npm run test:members` ensures all test profiles exist, have avatars, required frontmatter, and are listed in `authors.json`.
- `npm run build` to confirm the site still compiles.
- Optional: `npm run dev` and check the Team page carousel and categories (Faculty, Ph.D., M.Sc., Undergraduate, etc.) for your new entries.

## How categorization works
- `src/utils/authorLoader.ts` fetches `public/assets/data/authors.json` (falls back to a built-in list) to know which profile folders to load.
- The `user_groups` array in each profile drives grouping. The role string is a secondary fallback (contains `phd`, `m.sc.`, `research associate`, or `administrative assistant`).
- Ordering on the Team page is by `weight` (ascending) from the profile frontmatter.

## Common issues and fixes
- Member missing: ensure the folder name matches the ID in `authors.json`, `user_groups` is set, and the profile is listed in `authors.json`. Run `npm run members:sync-index` if you added a folder manually.
- Avatar not showing: confirm `avatar.png` exists in the member folder; the loader falls back to `default_avatar.png` if none is found.
- Broken fetch paths after deployment: always reference assets via `getAssetUrl`/`getContentUrl` (base-aware) as used in `authorLoader.ts`.
